OpenGL教學.01 什麼是OpenGL以及他的用處
2019-07-14|OpenGL


在這個網路流行的時代 大部分人都會去玩遊戲休閒娛樂，用自己的爪機 (個人數字助手 也就是現在的智慧機) 打遊戲 或者是錄製遊戲影片這種事情已經變得非常常見，這些遊戲有的畫面看起來很好 有的也比較一般，遊戲的畫面渲染依靠圖形介面實現(目前已知的有OpenGL，Direct3D，OpenGLES，Vulkan(新的圖形介面)還有以前Voodoo作死前比較流行的Glide) 。那麼我們先開始明白OpenGL是什麼 它能做什麼事情等。

什麼是OpenGL？

OpenGL (Open Graphics Library) 是一個開放圖形庫 用於提供基於二維或者三維畫面渲染的一個程式介面，並且在大部分 (幾乎全部或者大部分系統) 都有這個程式介面的支援 無論是行動電話，計算機，還是現在的智慧電視 都有OpenGL的支援 。OpenGL的標準在1992年由ARB (OpenGL Architecture Review Board，中文叫OpenGL架構評審委員會) 維護 直到2006年7月 ARB決定將接口標準的控制權利給Khronos團隊。之前有跟Direct3D做過競爭對手 有關更多的歷史介紹可以查閱Wikipedia的詳細說明。

OpenGL能做什麼事情？

OpenGL提供了很多用於圖形程式介面的基本渲染方法，和一些功能 讓立體物件渲染在畫面上。儘管OpenGL能做的事情有很多 但是也是要根據顯示卡的架構來決定。比如比較老一點的顯示卡ATI Radeon HD 4550 它的最高OpenGL版本在3.3 不具備大部分的OpenGL版本4的功能，再比如還有Intel GMA 3100 (G31/G33) 它的OpenGL支援更是低的不行 甚至連VBO Support (Vertex Buffer Object支援) 都沒有 直接導致了它只能使用固定渲染管線 並且沒辦法使用現在更主流的Core Profile (核心渲染管線)

OpenGL有兩種渲染管線 下面做個比較簡單的介紹

Fixed-Function Pipeline (固定渲染管線)

優點：簡單實現，對初學者非常友好，不需要特別複雜的功能就能簡單渲染立體模型或者二維畫面
缺點：自由度低，高頻率的處理器與顯示卡通訊會有機率Lag或者低FPS，Deprecated (已過期的並且已經被拋棄的 在3.0開始 直到3.3完全拋棄)

一種比較簡單的渲染管線 它的定義主要是爲了能夠簡單渲染立體模型，還有二維畫面 這種渲染管線在以前那個時候還比較流行 因爲它能夠跟Glide抗衡 但是自從DirectX的出現 這種渲染管線已經用的人不多了 也是因爲這個原因 就有了Core Profile的出現。

Core Profile Pipeline (核心渲染管線)

優點：效能不錯，可實現的功能更多，自由度高
缺點：對初學者不太友好，嚴重依賴Shader (渲染器)

比較複雜的渲染管線 它的主要定義是爲了能夠做更多的事情 使用Shader可以讓畫面更好看 更容易實現要求的功能。除此之外 它可以使用VBO (Vertex Buffer Object, 頂點緩衝區對象) 減少處理器跟顯示卡的高頻率通訊，靜態模型更是只要一次存入就足夠。

跟Direct3D的區別有多大？

在之前DirectX技術不是特別好的時候 Glide跟OpenGL完全就站在最高位 但是從DirectX 9版本提供了新的渲染方式的時候 OpenGL已經在DirectX後面了 主要也是因爲ARB咕咕咕 (才不是 是因爲內部出了點問題 導致員工不滿意等原因) 沒有提供新的功能 直到ARB把主要控制權交給了Khronos團隊的時候才開始有了一點點改變 目前Khronos團隊又提供了一個新的圖形程式介面 它的功能跟OpenGL相比更簡單 效能也比OpenGL強 這個東西叫Vulkan 等到鹹魚我有了支援Vulkan的顯示卡 還有相關技術書籍的時候再來討論。另外 Direct3D (包括DirectX) 只能在Windows上運作 而OpenGL是全平台支援，很明顯OpenGL在平台的受限是不存在的。

對OpenGL簡單的介紹就到這裏的 不知道作爲讀者的你有沒有興趣來試一試這麼強大的圖形程式介面？ 下一次我們將講解怎麼創建窗體 還有最基本的立體模型。
